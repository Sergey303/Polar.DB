<a id="specifications"></a>
# Термины и спецификации Polar.DB

---

<a id="specifications-terms"></a>
## Термины и определения

> **Структурное значение** (*structured value*, поляровское структурное значение, `p-value`) —
> древовидное построение, интерпретируемое в соответствии с заданным типом.

> **Тип** (*Type*, поляровский тип, `p-type`) — древовидное построение, задающее интерпретацию
> для структурных значений. Может быть выражено как структурное значение. Тип может быть
> примитивным (атомарным) или конструируемым (составным).

> **Объект базы данных** — сформированное в некотором рабочем поле или в хранилище
> структурное значение.

> **Объектное представление структурного значения** — внутренняя конструкция в ОЗУ, вместе с
> типовым значением задающая структурное значение. Объектное представление реализуется
> средствами некоторой системы программирования, например `.NET`, доступно через конкретный
> язык программирования и служит связующим средством между программными объектами и объектами
> базы данных.

> **Текстовая сериализация** — точные правила представления любого структурного значения
> последовательностью символов. Текстовое представление, совместно с явно или неявно
> определённым типом этого представления, есть структурное значение.

> **Бинарная (байтовая) сериализация** — правила представления любого структурного значения в
> виде потока байтов. Бинарная сериализация осуществляется только в контексте типа
> сериализуемого или десериализуемого структурного значения.

<a id="specifications-basic"></a>
## Начальные спецификации

<a id="specifications-primitive-types"></a>
### Примитивные типы

- `boolean` — логическое.
- `character` — символьное.
- `integer` — целое, 32 разряда.
- `longinteger` — длинное целое, 64 разряда.
- `real` — число с плавающей точкой, 64 разряда.
- `@byte` — байт, 8-разрядный код.
- `none` — множество значений, не содержащее ни одного элемента.
- `sstring` — системная строка, обычная строка объектно-ориентированного программирования (`C#`).

<a id="specifications-constructed-types"></a>
### Конструируемые типы

- **Запись** (`record`) — фиксированный набор типизированных полей. Есть поля `0, 1, ... n - 1`,
  каждое из которых представляет значение заданного в определении записи типа. Количество полей и
  типы полей фиксированы. Иногда полям сопоставляют имена-идентификаторы.
- **Последовательность** (`sequence`) — упорядоченный набор из неопределённого, но конкретного
  числа однотипных элементов.
- **Объединение** (`union`) — значение, состоящее из тега и подзначения. Тег динамически
  определяет вариант типа для подзначения. Теги нумеруются, начиная с `0`.

<a id="specifications-object-representation"></a>
### Объектное представление

Любое структурное значение может быть представлено в виде объекта по следующей схеме: примитивные
типы реализуются значениями соответствующих системных типов — логического, целого, длинного целого,
с плавающей точкой (`double`), символьного (`char`), байта, строки. Значения типа `none`
представляются значениями `null`. Конструируемые типы представляются массивами `object[]`,
элементами которых будут подзначения структур. Запись — массив объектных значений своих элементов,
последовательность — массив объектных значений элементов. Значение объединённого типа представляет
собой массив из двух элементов: первый элемент — целое значение тега, второй — объектное
представление подзначения соответствующего тегу типа.

#### Примеры

- Объектные значения `(object)22`, `(object)"demo string"`, `(object)true` могут быть
  интерпретированы только как целое, строка и логическое значение.
- Объект `new object[] { 2, 33 }` может быть интерпретирован, в зависимости от типа, как запись
  двух целых, как последовательность целых или как объединение с вариантом `2` и подзначением
  `33`.
- Для последовательности записей, состоящих из строкового и целого полей, объектным
  представлением может быть:

```csharp
new object[]
{
    new object[] { "str1", 111 },
    new object[] { "str2", 222 }
}
```

<a id="specifications-text-serialization"></a>
### Текстовая сериализация

Атомарные значения изображаются в виде текста: значение типа `none` — пустой строкой, значения типов
`boolean`, `character`, `integer`, `longinteger`, `real` — традиционно, как, например, в `C#`.
Значение `@byte` изображается шестнадцатеричным кодом, строка — как обычно, в `C#`.

Запись изображается перечислением через запятую значений всех полей записи, начиная с нулевого и
далее по порядку. Перечисление берётся в фигурные скобки. Возможно использование имён полей, как в
Паскале. Последовательность изображается перечислением элементов через запятую; всё перечисление,
нуль или более элементов, помещается в квадратные скобки. Объединение изображается тегом — числом в
диапазоне `0–255`, следующим за ним символом `^`, и далее идёт изображение значения того типа,
который динамически задан тегом. Значение может быть помещено в круглые скобки — это нужно для
определённости разбора. По структурному значению и его типу однозначно, с точностью до
несущественных синтаксических элементов, пробелов, перевода строки, `tab` и «лишних» скобок,
определяется текстовая развёртка. По корректной текстовой развёртке и типу однозначно определяется
структурное значение.

#### Пример

Для структурного значения типа последовательности записей строкового и целого полей,
заданного в объектном виде как

```csharp
new object[]
{
    new object[] { "str1", 111 },
    new object[] { "str2", 222 }
}
```

Текстовая сериализация будет:

```text
[ {"str1", 111}, {"str2", 222} ]
```

<a id="specifications-binary-serialization"></a>
### Бинарная сериализация

Бинарная сериализация выполняется следующим образом: атомарные значения отображаются на
последовательность байтов так, как это определяется системой программирования `C#` через метод
`System.IO.BinaryWriter` и двойственный ему `System.IO.BinaryReader`. Байт отображается в байт,
целое — в 4 байта, длинное целое — в 8 байтов, число с плавающей точкой — в 8 байтов, значение типа
`none` — в 0 байтов. Строка отображается сложнее: кодируются и количество символов, и сами символы в
кодировке `UTF-8`.

```csharp
byte[] info = new UTF8Encoding(true).GetBytes(str);
```

Запись сериализуется поставленными «встык» сериализованными значениями полей. Последовательность
вначале содержит 64-разрядное, 8-байтовое целое, фиксирующее количество элементов последовательности
(`0+`), а за ним подряд идут соответствующие значения элементов последовательности. Объединение
реализуется постановкой 1 байта, который фиксирует код тега (`0–255`); за этим байтом
непосредственно следует подзначение соответствующего тегу типа.

<a id="specifications-type-language"></a>
### Язык определения типов

Поляровские типы задаются некоторыми формулами, задающими типовые, ударение на первый слог,
значения. В принципе, эти значения можно рассматривать как обычные структурные значения, но эта
ветвь в текущем варианте Поляра не реализована. Для наглядности в иллюстрациях используется
специальный язык определения типов. Синтаксис языка в БНФ:

```text
типовое_определение: имя_типа = типовая_формула ;
типовая_формула: none
    | bool | byte | char | int | long | float
    | { (имя поля : )? типовая_формула,.. }
    | [ типовая_формула ]
    | имя_тега ^ типовая_формула
```

<a id="specifications-syntax-notes"></a>
### Синтаксические особенности и добавления

В формуле определения записи возможно использование имён полей. В случае, когда определяется
последовательность записей, фигурные скобки внутри прямоугольных можно опустить.
