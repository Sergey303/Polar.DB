# Состав депонируемых материалов

**Пометка:** ТОЧНО КОНЕЧНЫЙ ВАРИАНТ, кроме полей с пометкой «ПОПРАВИТЬ/ЗАПОЛНИТЬ».

## 1. Общая формулировка

В состав депонируемых материалов включены исходные тексты и иные материалы, позволяющие идентифицировать программу для ЭВМ «Polar.DB» версии 2.1.2.

Материалы относятся к конкретной версии программы: NuGet package `Polar.DB`, версия `2.1.2`.

## 2. Рекомендуемый состав исходных материалов

ПОПРАВИТЬ/ЗАПОЛНИТЬ после выгрузки версии 2.1.2 из репозитория/NuGet.

Рекомендуется включить:

1. Публичные API и основные классы библиотеки.
2. Описание и реализацию типов Polar / PType.
3. Реализацию сериализации и десериализации типизированных данных.
4. Реализацию последовательностей и операций чтения/записи.
5. Реализацию восстановления состояния, refresh/recovery-логики и работы с logical end / AppendOffset.
6. Реализацию индексов и поиска по ключу.
7. Реализацию RecordAccessor.
8. Файлы проекта, определяющие target frameworks и сборку NuGet-пакета.
9. Ключевые тесты, если решите включать их как дополнительные идентифицирующие материалы.

## 3. Что не обязательно включать

Можно не включать:

- временные файлы сборки;
- каталоги `bin/` и `obj/`;
- локальные benchmark-результаты;
- пользовательские тестовые базы данных;
- файлы `.db`, `.sqlite`, `.wal`, `.shm`, `.state`, `.index`, если они не являются частью исходного кода программы;
- секреты, токены, ключи публикации NuGet.

## 5. Проверка перед подачей

- Все материалы относятся именно к версии 2.1.2.
- Название программы в заявлении, реферате и титульном листе совпадает: «Polar.DB».
- Правообладатель указан одинаково во всех документах.
- Автор указан одинаково во всех документах.
- Размер программы совпадает в реферате, карточке заявки и титульном листе.
- Депонируемые материалы не содержат секретов.
