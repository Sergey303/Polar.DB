# GetStarted.SequencesAndStorage

Это второй учебный project в samples Polar.DB.

Он продолжает идеи из `GetStarted.StructuresAndSerialization` и переводит читателя от формы значения к поведению хранилища.

## Чему учит этот проект

- как записи хранятся в последовательностях;
- как выглядит append-ориентированный сценарий на практике;
- как связаны `Flush`, `Scan` и последовательный обход;
- как offsets могут использоваться для прямого доступа и вспомогательных lookup-структур;
- чем код, работающий с последовательностями, отличается от чисто in-memory примеров на `object[]`.

## Почему этот проект важен

Polar.DB — это не только схемы. Это ещё и надёжное структурное хранение.

Именно здесь читатель должен начать думать о таких вещах, как:
- логические границы данных;
- append-поток и read-поток;
- постоянная раскладка данных, а не только временные объекты в памяти.

## Предлагаемый порядок чтения

1. начать с простого примера на universal sequence;
2. затем пройти сценарии со scan и append;
3. после этого посмотреть сценарии с offset-ориентированным доступом;
4. и только потом переходить к отдельному tutorial по индексам и поиску.

## Типовые команды

```bash
dotnet run --project samples/GetStarted.SequencesAndStorage -- list
dotnet run --project samples/GetStarted.SequencesAndStorage -- all
dotnet run --project samples/GetStarted.SequencesAndStorage -- <scenario-id>
```

## Заметки

Этот проект должен оставаться сфокусированным именно на хранении и механике последовательностей.

Полноценные tutorial по индексам, token search и поиску по множественным значениям должны жить в `GetStarted.IndexesAndSearch`.
