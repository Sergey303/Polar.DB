# GetStarted.IndexesAndSearch

Это третий учебный проект в samples PolarDB.

Он посвящён актуальной поверхности lookup и search после того, как читатель уже понял типы, сериализацию, последовательности и базовые идеи хранения.

## Чему учит этот проект

- первичный поиск по ключу через `USequence`;
- вторичный точный lookup через `UIndex`;
- поиск по токену и по префиксу через `SVectorIndex`;
- точный поиск по множественным значениям через `UVectorIndex`;
- hash-based поиск по множественным значениям через `UVecIndex`;
- вспомогательные утилиты вроде `Scale`, `Diapason` и примеров hash-функций.

## Зачем нужен этот проект

Индексы проще понимать после того, как уже понятны основы хранения.

Этот проект нужен не только для абстрактной теории, а для того, чтобы увидеть, как реальные lookup-сценарии выражаются в коде PolarDB.

## Рекомендуемый порядок внутри проекта

1. сценарий с primary key;
2. secondary exact lookup;
3. text token и prefix search;
4. поиск по множественным значениям;
5. вспомогательные утилиты и поддерживающие примитивы.

## Типовые команды

```bash
dotnet run --project samples/GetStarted.IndexesAndSearch -- list
dotnet run --project samples/GetStarted.IndexesAndSearch -- all
dotnet run --project samples/GetStarted.IndexesAndSearch -- <scenario-id>
```

## Заметки

Этот проект должен оставаться практическим.

Примеры здесь лучше держать такими, чтобы они ясно показывали актуальный API и работали на маленьких наборах данных, которые легко проверить по выводу в консоль.
