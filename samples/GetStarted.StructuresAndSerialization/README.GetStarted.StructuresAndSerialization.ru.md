# GetStarted.StructuresAndSerialization

Это первый учебный проект в samples PolarDB.

Его задача — дать понимание базовых строительных блоков значений Polar до перехода к хранению и индексам.

## Чему учит этот проект

- как `PType` описывает данные;
- как записи, последовательности и другие структурные значения представляются в объектной форме;
- как `Interpret(...)` помогает смотреть на значение;
- как работают текстовая и бинарная сериализация;
- как schema-aware helper вроде `RecordAccessor` делает код понятнее и безопаснее.

## Рекомендуемый порядок чтения

Начинать лучше с самого простого сценария, затем переходить к более насыщенным примерам.

Хорошая последовательность такая:
1. базовое знакомство с записью и типом;
2. последовательность или более сложное структурное значение;
3. round-trip сериализации;
4. доступ к полям по имени.

## Что желательно понять после этого проекта

После прохождения этого проекта должно быть комфортно с такими идеями:
- у значения Polar всегда есть схема;
- объектное представление удобно, но всё равно управляется схемой;
- сериализация — не отдельная второстепенная тема, а нормальная часть работы с данными;
- доступ к полям по имени лучше, чем хрупкие магические индексы, когда важна структура записи.

## Типовые команды

```bash
dotnet run --project samples/GetStarted.StructuresAndSerialization -- list
dotnet run --project samples/GetStarted.StructuresAndSerialization -- all
dotnet run --project samples/GetStarted.StructuresAndSerialization -- <scenario-id>
```
