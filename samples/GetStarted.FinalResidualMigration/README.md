# GetStarted.FinalResidualMigration

Этот архив закрывает маленький остаток после основной раскладки `samples/GetStarted*` по тематическим проектам.

Что вошло:

- `Main1()` из `samples/GetStarted1/Program.cs` — перенесён как новый рабочий сценарий;
- старые shell-файлы `GetStarted2/3/4/Program.cs` — сохранены как historical reference;
- исходный `GetStarted1/Program.cs` тоже сохранён для сверки.

Почему так:

- содержательная предметная логика в остатке была по сути только у `Main1()`;
- остальные `Program.cs` в старых проектах были оболочками-переключателями между `Main...` сценариями;
- делать из них отдельные новые сценарии было бы искусственным дублированием.

Сценарии:
- `main1` — базовое знакомство с `PTypeRecord` и `PaCell`
- `shells` — печатает роль старых shell-файлов и зачем они оставлены в архиве

## Что всё ещё остаётся ручным решением

Если переносить всё в сам репозиторий, понадобится отдельно решить:

1. удаляются ли старые `GetStarted2/3/4/Program.cs` после миграции;
2. хранить ли их рядом как историческую справку;
3. нужно ли поднимать для `Main1()` отдельный тест.

## Содержимое historical / pending

- `HistoricalShells/` — сохранённые старые shell-файлы;
- `PendingMigration/` — исходный `GetStarted1/Program.cs` целиком для ручной сверки.
