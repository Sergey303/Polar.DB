# Поддержка

## Куда и с чем писать

### Сообщения о багах

Используйте GitHub Issues и выбирайте шаблон bug report.

Желательно указать:

- версию Polar.DB;
- target framework;
- операционную систему;
- минимальное воспроизведение;
- ожидаемое поведение;
- фактическое поведение;
- stack trace или логи, если они есть.

### Запросы на новую функциональность

Используйте GitHub Issues и выбирайте шаблон feature request.

Пожалуйста, опишите:

- какую проблему вы хотите решить;
- какой API или поведение предлагаете;
- какие альтернативы уже рассматривали;
- есть ли риск для совместимости.

### Вопросы по использованию

Пока такие вопросы тоже можно открывать в GitHub Issues с меткой `question`.

Если позже будет включён GitHub Discussions, общие usage questions лучше перенести туда.

### Вопросы безопасности

Не заводите публичный issue для уязвимостей.

Используйте приватный процесс из [SECURITY.md](SECURITY.md).

## Перед созданием issue

Проверьте:

- [README.md](README.md)
- [docs/Starting.md](docs/Starting.md)
- [docs/REPOSITORY_STATE.md](docs/REPOSITORY_STATE.md)
- уже существующие open и closed issues

## Примечание мейнтейнера

Сейчас Polar.DB находится в статусе experimental / pre-production. Поддержка оказывается по мере возможности.
