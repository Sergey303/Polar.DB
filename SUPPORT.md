# Support

## Where to ask what

### Bug reports

Use GitHub Issues and choose the bug-report template.

Please include:

- Polar.DB version;
- target framework;
- operating system;
- minimal reproduction;
- expected behavior;
- actual behavior;
- stack trace or logs, if available.

### Feature requests

Use GitHub Issues and choose the feature-request template.

Please explain:

- the problem you are trying to solve;
- the proposed API or behavior;
- alternatives considered;
- compatibility impact.

### Usage questions

For now, usage questions may also be opened as GitHub Issues and should be labeled `question`.

If GitHub Discussions is enabled later, general usage questions should move there.

### Security issues

Do **not** open a public issue for vulnerabilities.

Use the private reporting guidance in [SECURITY.md](SECURITY.md).

## Before opening an issue

Please check:

- [README.md](README.md)
- [docs/Starting.md](docs/Starting.md)
- [docs/REPOSITORY_STATE.md](docs/REPOSITORY_STATE.md)
- existing open and closed issues

## Maintainer note

Polar.DB is currently experimental / pre-production. Support is best-effort.
