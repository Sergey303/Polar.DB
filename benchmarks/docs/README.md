# Benchmarks

This folder documents the current benchmark layout.

## Layout

- `benchmarks/src/<Experiment>/Program.cs` contains one standalone experiment.
- `benchmarks/src/BenchSupport/` contains shared benchmark helper source files.
- `benchmarks/results/<experiment>.html` is generated by each experiment.
- `benchmarks/work/` contains temporary databases and files created during runs.
- `benchmarks/scripts/run-new-benchmarks.ps1` runs all current experiments.

The old benchmark runner, `experiment.json` workflow, old analysis/charts projects,
and `benchmarks/experiments` layout are no longer part of this benchmark system.

## Run

Run all experiments:

```powershell
powershell -ExecutionPolicy Bypass -File .\benchmarks\scripts\run-new-benchmarks.ps1
```

Run one experiment:

```powershell
dotnet run -c Release --project benchmarks\src\BuildPrimaryIntOnly\BuildPrimaryIntOnly.csproj
```

Run tests:

```powershell
dotnet test tests\Polar.DB.Tests\Polar.DB.Tests.csproj
```

Defaults live in `benchmarks/src/BenchSupport/BenchmarkDefaults.cs`.

Current row orders:

- `50_000`
- `5_000_000`

Current operation counts:

- normal lookup: `300` warmup operations, `2_000` measured operations;
- famous external lookup: `1` warmup operation, `3` measured operations;
- build-primary-int-only: `1` warmup run, `3` measured runs;
- reopen-only: `3` warmup runs, `20` measured runs;
- append/delete: `50` warmup operations, `2_000` measured operations.

## Build primary integer index only

`build-primary-int-only` measures only primary integer index build after data load.

For Polar.DB this opens `USequence` without external indexes and measures:

- `Sequence.Build()`;
- `Sequence.Flush()`.

For SQLite this loads rows into a table where `id` is not a primary key yet, then
measures creation of a unique index on `id`.

## Correctness

Correctness is checked against expected rows and checksums computed from the
generated dataset. Row order is ignored for correctness.
